## 环境：两条路，建议两条都用

验收会**暴露 bug → 我改 → 你重验**，所以要的是**快循环**；但有几项只有**装好的包**才暴露
（安装器 / 首启白屏 / 自启注册表 / 干净机 WebView2）。所以：

- **Track A · 本地 dev（主力，快循环）**：在 Windows 上按 [`BUILD-WINDOWS.md`](./BUILD-WINDOWS.md) 装好环境，
  跑 `pnpm tauri dev`。改一行我推给你，你 `git pull` 重跑，分钟级。**Phase 1–5 绝大多数用这个验。**
- **Track B · 出包（验装机 / 分发项）**：`pnpm tauri build` 出 `.exe`/`.msi`，或走 GitHub Actions
  [`release.yml`](../.github/workflows/release.yml)（Actions 页 Run workflow / 打 `v*` tag，三平台一次出）。
  **Phase 0 的装机项、Phase 6 的自启 / 托盘 / 干净机观感用这个验。**

> 没有本地 Windows 开发环境、只想拿包跑 → 全程用 Track B，但每次我改完要重新出包（CI ~10min/轮），
> 慢。能搭 Track A 就搭。
